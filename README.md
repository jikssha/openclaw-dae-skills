# openclaw-dae-skills
# OpenClaw 搜索技能集

这是一组面向 **OpenClaw** 的搜索技能仓库，目前包含两个技能：

- **`exa-search`**：偏 **源头检索 / 官方文档 / 低噪声结果 / 正文提取**
- **`grok-search`**：偏 **实时检索 / 舆论动态 / 多源综述 / 新鲜信息**

它们不是重复造轮子，而是专门解决两类完全不同的搜索问题：

1. **我需要找到真正的源头页面，最好还能直接抽正文**
2. **我需要知道“现在外面都在怎么说”，越快越好**

如果你只装一个搜索工具，大概率会在其中一类问题上吃瘪。  
所以这两个技能的设计目标不是互相替代，而是 **互补配合**。

---

## 这两个技能分别解决什么问题？

### 1）`exa-search`
`exa-search` 是一个 **source-first（源头优先）** 的搜索技能。

它更适合处理：
- 官方文档
- API 文档 / Reference 页面
- 产品说明页 / 定价页
- 公司官网 / 项目主页
- 需要抽取网页正文的场景
- 需要尽量减少 SEO 垃圾结果的场景

一句话：

> **当你要“找证据、找原文、找源头”时，用 `exa-search`。**

---

### 2）`grok-search`
`grok-search` 是一个 **freshness-first（时效优先）** 的搜索技能。

它更适合处理：
- 实时变化中的话题
- 社区讨论 / 舆论动态
- X / Twitter / 社区声量
- 新产品、新版本、新争议的快速综述
- “官方怎么说 + 社区怎么解读”的对比

一句话：

> **当你要“看现在发生了什么、大家怎么讨论”时，用 `grok-search`。**

---

## 为什么不直接只用一个通用搜索工具？

因为“搜索”本来就不是单一问题。

### 通用搜索工具常见的两个毛病

#### 毛病 1：结果很杂
你明明想要的是官方文档，结果给你一堆：
- SEO 博文
- 搬运站
- 二次总结
- AI 改写页面

#### 毛病 2：对新鲜信息反应太慢
你明明想知道“现在外面怎么说”，结果它只会给你：
- 老页面
- 过时资料
- 没有上下文的零碎链接

所以更合理的方案不是“一个工具包打天下”，而是：

- 用 **`exa-search`** 管 **证据质量**
- 用 **`grok-search`** 管 **信息时效**

---

## 什么时候选 `exa-search`，什么时候选 `grok-search`？

### 用 `exa-search` 的典型问题
- OpenClaw 官方文档里怎么写的？
- 这个 API 参数到底是什么？
- 这个产品官网的定价页在哪里？
- 帮我找源页面并抽取正文
- 只要官方 / 原始来源，不要二手总结

### 用 `grok-search` 的典型问题
- 现在大家怎么看这个产品？
- 最近外面在讨论什么？
- X 上关于这个话题有什么动静？
- 帮我快速总结近期讨论
- 对比一下官方说法和社区解读

---

## 如果只能二选一，该装哪个？

### 选 `exa-search`，如果你主要做的是：
- 查官方文档
- 查 API
- 查定价页 / 说明页
- 做工程研究
- 做 source-of-truth 验证

### 选 `grok-search`，如果你主要做的是：
- 跟踪热点
- 看社区反馈
- 追实时动态
- 看产品口碑 / 舆情
- 快速了解一个正在变化的话题

---

## 最推荐的用法：两个都装

这才是最强组合。

### 推荐工作流
1. 先用 **`grok-search`** 看外部世界现在在发生什么
2. 再用 **`exa-search`** 去找官方链接和源页面做验证

这样你能同时拿到：
- **快**：快速理解局势
- **准**：回到官方和源头核实

一句话：

> **`grok-search` 帮你找方向，`exa-search` 帮你找证据。**

## 安装方式

把对应 skill 文件夹复制到你自己的 OpenClaw `skills/` 目录下即可。

例如：

```text
skills/
├── exa-search/
└── grok-search/
```

---

## 配置说明

仓库中 **不包含真实 API Key**。

你需要根据每个 skill 里的示例配置，自己创建本地配置文件：

```text
config.example.json -> config.local.json
```

然后填入你自己的 key。

## `exa-search` 特性

### 核心能力
- 官方文档友好
- 支持正文提取
- 支持 `docs / research / similar` 预设
- 支持多 key / 多 profile 自动 failover
- 支持 `--plain` / `--urls`

### 适合谁
- 独立开发者
- 运维 / 工程师
- 需要经常查文档的人
- 想尽量避开搜索噪声的人

---

## `grok-search` 特性

### 核心能力
- 支持 `general / news / social / research / docs-compare` 模式
- 支持多 key / 多 profile 自动 failover
- 支持 cooldown，避免坏 key 被反复撞击
- 支持 `--plain` / `--urls`
- 支持“官方说法 vs 社区解读”结构化输出

### 适合谁
- 想跟踪实时舆论的人
- 想看社区动态的人
- 想快速把一个话题摸清轮廓的人
- 想把“外部讨论”纳入研究流程的人

---

## 安全说明

这个仓库默认不提交：
- `config.local.json`
- runtime 状态文件
- cooldown 状态文件
- 任何真实 API key

请不要把你自己的本地密钥提交到 GitHub。

仓库中的 `.gitignore` 已经帮你挡了一层，但别犯低级错误。
